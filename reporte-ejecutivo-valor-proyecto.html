Vicepresidencia de Negocio y Tecnología • Marzo 2026

Resumen Ejecutivo

El proyecto prueba-agente-po es un workspace integrado que centraliza la calidad de software, la automatización de pruebas y la observabilidad del ecosistema Ciencuadras. Entrega valor mediante: (1) detección temprana de errores en producción mediante auditorías automatizadas, (2) validación continua de flujos críticos con tests E2E, (3) documentación operativa para respuesta a incidentes, y (4) orquestación de agentes de IA para acelerar el ciclo de desarrollo con trazabilidad a Jira y Datadog.

Valor Entregado

🛡️ Calidad Proactiva

Tests E2E automatizados (Home, Arriendo, Venta) y validación de ausencia de errores críticos de consola. Reduce riesgo de regresiones antes de producción.

🔍 Auditoría de Consola

Script que navega 9 zonas críticas de ciencuadras.com, captura errores/warnings de JavaScript y genera reportes con capturas de pantalla para priorización.

📊 Observabilidad Documentada

Runbooks operativos, SLOs recomendados, mapeo servicio-repositorio y especificación de dashboards en Datadog. Acelera la respuesta a incidentes.

🤖 Orquestación con IA

Reglas de agente (Scout, Historian, Guardian) que integran Jira, GitHub y Playwright. Planificación en .cursor/plans antes de cambios de código.

Herramientas Utilizadas y su Valor

🎭
Playwright

Tests E2E y Auditoría

Framework de automatización para smoke tests en flujos críticos y script de auditoría de errores de consola. Genera reportes HTML y capturas de pantalla.

📋
Jira (MCP Atlassian)

Gestión y Trazabilidad

Integración vía MCP para leer tickets, extraer requerimientos y vincular entregas. Permite al agente Scout contextualizar tareas desde el backlog.

📈
Datadog (MCP)

Monitoreo y Observabilidad

Consultas a logs, métricas, monitores y traces. Documentación de SLOs, runbooks y dashboards para Ciencuadras. Soporta decisiones basadas en datos.

🔧
ESLint + Prettier

Calidad de Código

Estándares de linting y formato en el proyecto raíz. Mantenibilidad y consistencia del código de pruebas y scripts.

🤖
Cursor Rules / Agentes

Orquestación de IA

Reglas que coordinan fases: Análisis (Jira), Contexto (GitHub), Planificación (.cursor/plans), Validación (Playwright). Evita mezcla de contextos.

Impacto en Negocio y Tecnología

  • Reducción de incidentes: Detección temprana de errores de consola y validación de flujos críticos antes de afectar usuarios.
  • Respuesta más rápida: Runbooks y documentación de observabilidad acortan el tiempo de diagnóstico en incidentes.
  • Visibilidad de SLOs: Propuesta de objetivos de nivel de servicio permite reportar disponibilidad con datos reales.
  • Aceleración del desarrollo: Orquestación de agentes con trazabilidad a Jira y validación automática con Playwright.
  • Estandarización: Calidad de código (ESLint/Prettier) y protocolos de ejecución para agentes.
